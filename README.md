# sql-projects

📌 Project Overview
This project analyzes a Customer–Orders relational database using SQL. It focuses on customer behavior, order trends, revenue insights, and overall business performance. The database simulates a small e-commerce system and includes multiple analytical queries.

🗂️ Database Structure
Customers Table:
Stores customer information such as name, city, email, and registration date.

Orders Table:
Contains details of each order including amount, order date, customer reference, and order status (Completed, Pending, Cancelled).

🔍 Key Analysis Performed
Retrieve customers based on location
Identify high-value orders
Count orders per customer
Calculate total revenue
Filter orders by date range
Find highest spending customers
Identify customers with multiple orders
Retrieve first and most recent orders
Analyze order status distribution
Identify customers with completed orders

🛠️ SQL Concepts Used
JOINs (INNER JOIN)
GROUP BY with aggregations (SUM, COUNT, AVG)
ORDER BY & LIMIT
Subqueries
Date filtering (BETWEEN, YEAR())
Aggregate analysis
DISTINCT usage
HAVING clause

📊 Insights
Certain customers place multiple orders, contributing more to sales
Completed orders generate the highest total revenue
Order activity is clustered around specific dates
Some customers have very high-value orders
Revenue distribution across statuses shows business performance trends

🚀 Tools Used
MySQL / MariaDB
